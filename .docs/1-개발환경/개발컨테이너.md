# 개인 개발 컨테이너

개발 PC(Windows 11)에서 WSL2 없이 **GPU·CUDA·ODBC·Node·Python 을 한 컨테이너에 묶어** SSH 로 접속하는 구성.

## 구성 요약

| 항목 | 내용 |
| --- | --- |
| 베이스 이미지 | `nvidia/cuda:12.8.1-cudnn-devel-ubuntu24.04` |
| Python | uv 0.10.11 / Python 3.12 (`/opt/uv/`) |
| Node.js | 24.16.0 LTS |
| GPU | NVIDIA CUDA 12.8 (cudnn 포함) |
| DB 드라이버 | msodbcsql18 (MS SQL ODBC) |
| Java | OpenJDK 17 (MLflow 등 의존) |
| 개발 도구 | Claude Code · rtk · process-compose · pre-commit · ruff |
| SSH 포트 | 호스트 `10422` → 컨테이너 `22` |
| App 포트 | `8000` (backend-service) |
| 리소스 | CPU 16코어 · RAM 32 GB · GPU 1장 |

## docker-compose 주요 설정

```yaml
container_name: workspace_kch
ports:
  - 10422:22
  - 8000:8000
volumes:
  - ./volume/kch/root:/root         # 홈 디렉토리 (클로드 설정·SSH 키 등 영속)
  - ./volume/kch/data:/home/volume  # 작업 디렉토리
  - mlflow-mlruns:/mlruns:ro        # MLflow 아티팩트 공유 (읽기 전용)
networks:
  - kafka                           # kafka 브로커 컨테이너와 같은 네트워크
deploy:
  resources:
    limits: { cpus: '16.0', memory: 32G }
    reservations:
      devices: [ { driver: nvidia, capabilities: [gpu] } ]
shm_size: "8G"
```

## 시작 / 접속

```bash
# 기동 (호스트 머신)
docker compose up -d workspace_kch

# 컨테이너 내 서비스 일괄 기동
process-compose up
```

## SSH 키 인증 설정

비밀번호 없이 `ssh workspace` 한 줄로 접속하려면:

### 1. 로컬 PC 에서 키 생성 (없으면)

```bash
ssh-keygen -t rsa -b 4096   # ~/.ssh/id_rsa + id_rsa.pub 생성
```

### 2. 공개키를 컨테이너 루트 홈에 등록

```bash
# 최초 1회 — 비밀번호로 접속해서 복사
ssh-copy-id -p 10422 root@<server-ip>
# 또는 수동으로
cat ~/.ssh/id_rsa.pub | ssh -p 10422 root@<server-ip> "cat >> /root/.ssh/authorized_keys"
```

컨테이너 볼륨 `./volume/kch/root:/root` 가 영속이므로 재시작해도 키가 유지된다.

### 3. 로컬 `~/.ssh/config` 에 별칭 추가

```text
Host *
    IdentityFile ~/.ssh/id_rsa
    ServerAliveInterval 60
    ServerAliveCountMax 3
    StrictHostKeyChecking no
    UserKnownHostsFile /dev/null
    Compression yes
    LogLevel ERROR

Host workspace
    HostName <server-ip>
    Port 10422
    User root
```

이후 `ssh workspace` 로 바로 접속. VSCode Remote-SSH 도 동일 별칭 사용.

## Git 원격 SSH 설정 (컨테이너 내부)

컨테이너 안에서 `git clone git@<git-host>...` 로 당기려면 컨테이너 자체 키를 Git 호스트 에 등록해야 한다.  
볼륨 `./volume/kch/root:/root` 가 영속이므로 키는 한 번만 만들면 된다.

### 1. 컨테이너 내부에서 키 생성

```bash
ssh-keygen -t ed25519 -C "workspace" -f ~/.ssh/id_ed25519
# 패스프레이즈 없이 Enter 두 번
```

### 2. 공개키를 Git 호스트(GitHub 등)에 등록

```bash
cat ~/.ssh/id_ed25519.pub
```

복사 후 Git 호스트 → **SSH Keys → Add new key** 에 붙여넣기.

### 3. 연결 테스트

```bash
ssh -T git@<git-host>
# Welcome to GitHub, @username! 이 나오면 성공
```

### 4. clone

```bash
git clone git@<git-host>:<group>/<repo>.git
```

> Git 호스트가 자체 호스팅(`git.example.com` 등)이면 `~/.ssh/config` 에 호스트를 추가해 포트를 맞춘다.
>
> ```text
> Host <git-host>
>     HostName <git-host>
>     User git
>     IdentityFile ~/.ssh/id_ed25519
>     Port 22          # 기본 22, 변경 시 수정
> ```
