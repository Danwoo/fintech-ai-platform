# fintech-ai-platform — 발전 로드맵

> 포트폴리오 센터피스(은행·증권 타깃). "플랫폼 → 응용"을 하나의 앱으로 증명한다.
> (내부 계획 문서 — 공개 보류)

## 현재 상태

- **13서비스 MSA** — backend-service · frontend(Next.js, 332파일) · platform · 7개 MCP 서버 + multi-agent / single-agent / devactivity / file
- **에이전트** — Plan-Execute 멀티에이전트 (4도메인 · 12 sub-agent)
- **MCP** — 7 서버 · `from_fastapi` 27곳 · operation_id(=MCP tool) 52개 · lockstep 검증됨
- **인프라** — compose(prod/staging/process) · Dockerfile 14 · nginx · 멀티테넌트 · 가드레일
- **문서** — `.docs` 28개 설계문서(아키텍처·기법·개발가이드) · `.claude` 패턴 SoT(design/anti-pattern)
- **데이터** — 현재 **mock**. MCP lockstep · AST · leak 0 검증 완료.

## 강점

유일한 포트폴리오 앱이자 센터피스. 인프라 + 코드패턴 + 설계사상 + MCP 철학을 *하나에* 담아, "플랫폼부터 응용까지"를 코드로 자체 증명한다.

## 갭 / 한계 (정직하게)

1. **자체 코드가 아직 P0 리뷰를 안 거침** — 추출 라이브러리들만 코드리뷰·리팩토링했고, 정작 플래그십은 미검증. mock이 가린 버그·lockstep 불일치·누수가 남아 있을 수 있음.
2. **mock 데이터** — 재현 가치 낮음. 실데이터 전환 필요.
3. ~~env-특정 잔재 (개인 도구·이니셜 언급)~~ — **정리 완료** (일반 도구명으로 치환, 내부 이니셜/마운트 경로 제거).
4. **계보 서사 부재** — 어떤 패턴이 어느 참고 조각으로 증류됐는지 명시 문서 없음.

## 로컬 발전 스텝 (우선순위)

1. **자체 코드 리뷰 · 리팩토링** ★최우선 — 라이브러리에 적용한 방식(P0 정확성/보안/lockstep → 검증 → 수정) 그대로. **키 없이 로컬에서 지금 가능.** 버그 있는 채로 실데이터를 얹지 않기 위해 선행.
2. **실데이터화 (집에서)** — 토스증권(시세 + 실 계좌/보유) · DART(공시·재무제표) · 네이버(뉴스). `USE_REAL_API` 토글 뒤에 실연동, 오프라인 데모는 실 샘플 번들. "한 경로를 완전 실데이터로 → 확장" 순서, **앱 전체**. 주문 API는 조회·dry-run만(실주문 ✗).
3. ~~env-특정 정리 (공개 전)~~ — **완료** (개인 도구명·이니셜 일반화).
4. **계보 서사 문서** — 앱의 어떤 패턴이 어느 참고 구현으로 나왔는지 (면접 대비).

## 은행·증권 각도

- **공시(DART)** = 리서치의 심장 (disclosure-mcp)
- **실 계좌(토스)** = 실제 포트폴리오 분석 (portfolio-mcp — 합성 → 진짜)
- **가드레일** = 컴플라이언스(투자조언 아님 고지 · 미근거 수치 방어)
- **KYC/계좌서류** = document-ai(참고 구현) 연계 여지

## 결정 로그

- **자기완결 프로덕트로 유지** — 라이브러리를 import·조립하는 구조로 바꾸지 않음(억지 wrapping 회피). 라이브러리와의 관계는 코드 의존이 아니라 **계보(distilled from)**.
- **두 에이전트 패턴 병존** — 앱의 Plan-Execute(다단계 계획)와 참고 구현 agentic-rag(라우팅+병렬툴)는 별개 패턴으로 둔다.
- **앱 전체 실데이터화** — vertical slice 아님.
- **소수 정예** — 포트폴리오로 유지·전시하는 유일한 앱 (+ tabular-ml-kit 신용/리스크). 나머지는 비공개 참고 보관.
